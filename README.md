## ✍️ Adding a Blog Post or Poem

1. Create a `.txt` file inside the `blogs/` folder.
2. Name it however you like, e.g. `003.txt`, `rainy-afternoon.txt`.
3. Use this format:

```
Title: Your Title Here
Date: 2024-03-15
Type: poetry

---

Your content starts here.

New paragraph after a blank line.

For poetry, every line break is preserved.
```

- `Type` can be `poetry` or `blog`
  - **poetry** → italic, large text, line breaks preserved
  - **blog** → prose paragraphs

4. **Push to GitHub.** The Action will automatically update `blogs/index.json`.

## 📷 Adding Photos

### Cover photo for a post
Name the photo the **same as the blog file**:
- `blogs/003.txt` → `photos/003.jpg`
- `blogs/rainy-afternoon.txt` → `photos/rainy-afternoon.jpg`

### Gallery-only photo
Name it anything that does **not** match a blog file:
- `photos/cityscape.jpg` → appears in Gallery only
- `photos/power-lines.jpg` → Gallery only

Supported formats: `.jpg`, `.jpeg`, `.png`, `.webp`

Push to GitHub — the Action updates `photos/index.json` automatically.

---

## ⚙️ How the Auto-Index Works

The file `.github/workflows/update-index.yml` is a GitHub Action that runs
automatically whenever you push new `.txt` or photo files.

It:
1. Scans `blogs/*.txt` → writes `blogs/index.json`
2. Scans `photos/*.(jpg|jpeg|png|webp)` → writes `photos/index.json`
3. Commits and pushes the updated JSON files

**You never need to edit the index files manually.**

## 🎨 Customising

- Edit the **About** section text in `app.js` → `renderAbout()` function
- Change colours in `style.css` → `:root` CSS variables
- Change fonts by updating the Google Fonts link in `index.html`

## 🚀 First-Time Setup

1. Create a GitHub repository named exactly `ashik0025.github.io`
2. Push all these files to the `main` branch
3. Go to **Settings → Pages** → Source: `Deploy from branch` → Branch: `main` / `/ (root)`
4. Your site will be live at `https://ashik0025.github.io` within a few minutes
